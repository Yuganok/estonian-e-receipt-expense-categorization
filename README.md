# E-Receipt Ledger (Desktop)

Desktop app for downloading and analyzing grocery e-receipts in Estonia.
The Electron UI orchestrates a Python pipeline that parses receipt lines, assigns expense categories, optionally matches receipts to bank transactions, and generates report artifacts.

## What the app does

- Downloads Maxima receipts for a selected date range (via browser login flow).
- Runs receipt parsing and rule-based categorization.
- Supports two analysis modes:
  - **Full mode**: standard pipeline over `receipts/` (bank CSV optional).
  - **Single receipt mode**: analyze one manually selected text-PDF receipt (for example, Selver).
- Stores each run in `output/sessions/<timestamp>/` with CSV outputs and research artifacts.

## Prerequisites

- Node.js + npm
- Python 3 (Windows launcher `py` is expected by the desktop app)

Python dependencies are defined in `analysis/requirements.txt`:

- `pypdf`
- `scikit-learn`

## Installation

From project root:

```bash
npm install
```

Install desktop UI dependencies:

```bash
cd desktop
npm install
cd ..
```

Install Python dependencies:

```bash
py -3 -m pip install -r analysis/requirements.txt
```

## Run the desktop app

```bash
cd desktop
npm start
```

The UI has four sections:

- **Data Collection**
- **Parsing Review**
- **Categorization Review**
- **Baseline & Reporting**

## Typical usage

### A) Full mode (standard analysis)

1. Open **Data Collection** tab.
2. Set date range and run receipt download (Maxima).
3. Keep **Analysis mode = Full pipeline**.
4. (Optional) Pick a bank CSV via **Browse CSV**.
5. Run analysis.

Behavior:

- If bank CSV is selected, receipt-to-bank matching is performed.
- If bank CSV is not selected, matching is automatically skipped and a compatible `matched.csv` stub is generated.

### B) Single receipt mode (Selver-friendly flow)

1. Set **Analysis mode = Single receipt**.
2. Choose one text-PDF receipt via **Browse PDF**.
3. (Optional) Set store label, e.g. `Selver`.
4. Run analysis.

Behavior:

- No bank CSV is required.
- Only the selected PDF is analyzed.
- Bank matching is skipped automatically.

## Input and output paths

Main folders in repository root:

- `receipts/` - downloaded or manually placed receipts
- `bank_exports/` - optional bank CSV files
- `output/sessions/<timestamp>/` - run-specific outputs

Common output files per run:

- `items_raw.csv` - parsed item rows
- `items_categorized.csv` - categorized item rows
- `receipts.csv` - one row per receipt
- `matched.csv` - bank matching output (or stub if skipped)
- `category_breakdown.csv`
- `category_source_breakdown.csv`
- `report.txt`

Research artifacts (when generated by the app):

- `research/run_manifest.json`
- `research/quick_metrics_definition.txt`
- `research/manual_corrections.csv` (if saved from UI)

## Troubleshooting

- **`py` command not found**
  - Install Python 3 for Windows and ensure Python launcher (`py`) is available in PATH.
- **Invalid PDF/CSV path**
  - Use absolute file paths selected from UI pickers (`Browse PDF`, `Browse CSV`).
- **No item rows extracted from single receipt**
  - Ensure PDF has a readable text layer (not only scanned image).
- **Unicode/path issues on Windows**
  - Prefer selecting files from the UI instead of typing paths manually.

## Notes and limitations

- Maxima receipt download is supported from the app.
- Selver receipts can be analyzed in **Single receipt mode**, but **automatic Selver download is not available**.
